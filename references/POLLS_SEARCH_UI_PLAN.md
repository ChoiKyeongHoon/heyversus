# polls 페이지 검색/선택 전용 UI 플랜

## 목표
- `/polls`를 “투표 검색/선택” 허브로 전환하고, 페이지 내 직접 투표 기능을 제거한다.
- 로그인/비로그인 상태 모두에서 이미 참여한 투표/미참여 투표를 명확히 구분 표시한다.
- 투표는 상세 페이지(`/poll/[id]`)에서만 진행하도록 안내한다.

## 작업 범위
- **투표 기능 제거**: 목록 카드/CTA에서 옵션 선택·투표 버튼을 제거하고, “투표 하러 가기” 링크만 유지.
- **상태 배지/UX**: `has_voted` 여부(로그인/비로그인 공통)와 마감 상태를 카드 배지/텍스트로 노출해 참여 여부를 한눈에 알 수 있게 한다.
- **검색/필터 강화**: 기존 카테고리/정렬/필터를 재배치해 “보고 싶은 투표 찾기” 경험에 집중. 필요 시 검색 입력/태그 필터 추가.
- **상세 진입 동선**: 카드 CTA와 메타 copy를 “상세에서 투표하세요”로 통일해 흐름을 명확히 한다.
- **시각 톤**: PollsHero/카드 토큰 스타일을 유지해 상세 페이지와 일관성 확보.

## 참고/후속
- 상세 페이지 UI 일원화(ROADMAP Step 17) 이후 동일 컨벤션을 `/polls`에도 적용.
- 투표 여부 표시는 현재 `has_voted` + 익명 로컬 스토리지 표준(`useVoteStatus`)을 그대로 활용.
